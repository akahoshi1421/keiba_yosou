## 現状
現状は複勝と三連複を買うアルゴリズムになっているはずです。

しかし、三連複はある程度の馬券を買わないと当たらないので回収率が非常に悪いです。

## 提案
そこで三連複馬券を買うことを「「「廃止」」」とし。以下の馬券を買うこととしてください。

- 単勝
- 複勝
- ワイド

それぞれ買う馬券の枚数は以下の通りです。

- 単勝 => 2枚
- 複勝 => 2枚
- ワイド => 8枚

## 評価方法
従来は三連複馬券で勝率20%を超えるまで「アルゴリズムを見直す+修正をする+再シミレーションを行う」を繰り返すこととしていました。

しかし、今回は以下のようにします。

```
ワイド馬券で勝率50%を超えるまで「アルゴリズムを見直す+修正をする+再シミレーションを行う」を繰り返す
```

ただし、今回は勝率の定義が異なります。ワイドは1レース最大3枚的中するためです。

その「勝率」を求める計算式は以下です。
```
(総的中ワイド馬券数 / (評価対象レース数 * 3)) * 100 = パーセント勝率
```

gemini.tsに記載しています。(これはあなたへの指示をプログラム風に落とし込んだものです。このプログラムの通りに評価を行っていってください。)
