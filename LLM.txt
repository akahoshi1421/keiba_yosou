競馬予想は以下のアルゴリズムで考えたいです。
概要を書くと過去の戦績や人気順、当該馬の親の戦績をもとに加算方式で評価を行い。一番数値が高い馬を期待値が高い馬として提示するものとしてください。

また、この競馬予想プログラムは「三連複」を予想するものです。

数字が高い馬をピックアップして10枚ほどおすすめ馬券を提示してください。(提示は馬番号でお願いします。)

また、このプログラムは将来的に北九州記念以外にも使いまわしたいです。それを意識したコード作成をしてください。

以下が評価の概要です。

## 評価の概要
### 過去の戦績
過去の馬のレースを取得し、着順が高ければ高いほど大きな数値を与えてください。
ただし、着差が少なければ少ないほど加算される数値は差を生みにくくなるようにしてください。

例えば2着でゴールインし、1着との着差がそれほどないのであればそれぞれに加算される数値はほぼ同一(若干1着の方が多い)にし、逆に1着と2着の差が大きければ数値をその分開けてください。

さらに着順番号が高い上に今回のレースと性質が似ている競馬場(距離や芝、天候等)であれば、似ていれば似ているほどさらに別個で数値を加算してください。

### 人気順
今回のレースで人気が高ければ高いほど大きな数値を与えてください。



### 親馬の戦績
親馬も同様に過去の戦績と人気順の評価を行ってください。ただし、加算される数値は`出走馬がその条件で加算される時のスコア/2`としてください。


## 技術的な仕様
### 当該馬のデータの取り方
当該馬のデータを取得したい場合は以下から取得してください。
あなたが先ほど取得した出馬表APIのhourse_idがURLとなっています。

たとえば、hourse_idが2016104668の場合は

```
https://db.netkeiba.com/horse/2016104668
```

となります。

### 親馬の取り方
当該馬のページに親馬の情報があります。

例えば上に提示した馬であれば親馬は「ジャスタウェイ」と「レイズアンドコール」です。
その親馬が書かれているテキストはリンクになっています。

例えばジャスタウェイは
```
https://db.netkeiba.com/horse/ped/2009106461/
```
です。

なのでリンクを以下のように書き換え、出走馬と同じように分析してください。(ただし、それより親馬はたどらないでください。)
```
https://db.netkeiba.com/horse/2009106461
```

Good luck！
